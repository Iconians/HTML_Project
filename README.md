**Project: Build a Complete Multi-Page Website**

Create a fully functional, multi-page website that demonstrates mastery of all HTML concepts learned throughout this course.

**Reminder: Create a repo (GitHub + local)**

- **On GitHub:** Sign in → "+" → "New repository" → Name it (e.g. `html-project`) → Leave "Add a README file" **unchecked** → Create repository.
- **In the terminal (locally):**
  1. `mkdir project-name` then `cd project-name`
  2. `git init`
  3. Create your files, then: `git add .` → `git commit -m "Initial commit"`
  4. `git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git`
  5. `git branch -M main` (if needed)
  6. `git push -u origin main`

**Requirements:**

**Structure (25%):**
- Minimum 4 HTML pages (homepage, about, services/portfolio, contact)
- Proper folder organization
- Semantic HTML throughout
- Consistent navigation between pages

**Content (25%):**
- Proper heading hierarchy (h1-h6)
- Paragraphs, lists, and text formatting
- Images with alt text
- Links (internal, external, anchor links)
- Forms (contact form with validation)

**Advanced Features (20%):**
- Meta tags (description, viewport, Open Graph)
- Tables (if appropriate for content)
- Media elements (video/audio or iframe)
- HTML5 features (details/summary, time, etc.)

**Accessibility (15%):**
- All images have alt text
- Forms have proper labels
- Semantic HTML used
- Keyboard navigation works
- Proper heading structure

**Code Quality (15%):**
- Valid HTML5
- Clean, organized code
- Consistent formatting
- Comments where helpful
- Best practices followed

**Submission:**
Submit all HTML files, organized in a proper folder structure. Include a README explaining your project.

**Grading Criteria:**
- Completeness and functionality
- Code quality and best practices
- Accessibility implementation
- Use of semantic HTML
- Overall project execution
